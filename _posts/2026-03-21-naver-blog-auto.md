## 프로젝트 개요

블루오션 키워드 발굴 → Vision 이미지 분석 → LLM 초안 생성 → D.I.A.+ 10개 항목 SEO 검증 → 4가지 전략 자동 최적화 → 원클릭 발행까지 전 과정을 자동화하는 1인 사용자용 블로그 도구이다. 로컬 LLM(Ollama)을 사용하여 API 비용 없이 운영할 수 있다.

> 소스 코드는 요청 시 제공 가능합니다.

## 핵심 문제와 해결 접근

### 문제 1: 경쟁이 낮은 키워드를 어떻게 찾을 것인가

블로그 상위 노출의 핵심은 "검색 수요는 있지만 경쟁이 적은" 블루오션 키워드를 찾는 것이다. 수동으로 키워드를 탐색하면 시간이 오래 걸리고 감에 의존하게 된다.

**해결: 네이버 자동완성 + LLM 확장 + 블루오션 점수 산출**

```
시드 키워드 입력
    │
    ├── 네이버 자동완성 API 크롤링 → 실제 검색 수요가 증명된 키워드
    ├── LLM 롱테일 확장 → 15개 변형 키워드 생성
    └── 중복 제거 + 소스 태깅 (자동완성 우선)
    │
    ▼
블루오션 점수 산출 (0~100점)
    │
    ▼
상위 포스트 품질 분석 (경쟁글 평균 길이, 이미지 수, 품질 등급)
```

**블루오션 점수 알고리즘** (3가지 지표의 가중합):

| 지표 | 배점 | 산출 방식 |
|------|------|----------|
| **소스 점수** | 30점 | 자동완성 출처 = 30점 (검색 수요 증명), LLM 생성 = 10점 |
| **상대 경쟁도** | 40점 | `seed_ratio = 타겟 블로그 수 / 시드 블로그 수` (로그 스케일) |
| **구체성** | 30점 | 시드 대비 추가 단어 수 (3개 이상 = 30점, 2개 = 25점) |

`seed_ratio`가 핵심이다. 단순 블로그 수가 아닌 **시드 키워드 대비 상대적 경쟁 수준**을 측정한다:

| seed_ratio | 점수 | 판정 |
|-----------|------|------|
| 1% 미만 | 40점 | 강력한 블루오션 |
| 1~5% | 35점 | 블루오션 |
| 5~15% | 28점 | 준 블루오션 |
| 15~30% | 18점 | 중간 경쟁 |
| 30~60% | 10점 | 높은 경쟁 |
| 60% 이상 | 3점 | 레드오션 |

예를 들어 "강남 카페"가 50,000건이고 "강남 카페 조용한 노트북"이 200건이면 `seed_ratio = 0.4%`로 블루오션(40점)에 해당한다.

키워드 선택 시 **상위 포스트 품질 분석**도 함께 제공한다. 경쟁글의 평균 길이, 이미지 수, 품질 등급(높음/중간/낮음)을 표시하여 진입 난이도를 직관적으로 판단할 수 있다.

### 문제 2: SEO 점수를 정량적으로 진단하고 자동 개선할 수 있는가

네이버 검색에서 상위 노출되려면 D.I.A.+ 알고리즘의 기준을 충족해야 하는데, 이 기준은 공식적으로 공개되지 않는다. 블로거들은 감으로 글을 쓰고 결과를 확인할 수밖에 없었다.

**해결: D.I.A.+ 10개 항목 검증 + 3가지 프로파일 + 4가지 전략 자동 개선**

리서치를 통해 D.I.A.+ 알고리즘의 핵심 지표를 10개로 정리하고, 각 지표에 대해 정량적 검증 로직을 구현했다:

#### 10개 SEO 검증 항목

| 항목 | 검증 기준 | 비고 |
|------|----------|------|
| **제목** | 25자 이내, 키워드 앞쪽 배치 | 기본 |
| **본문 길이** | 최적 2,500~3,000자 (최소 1,500자) | 기본 |
| **키워드 밀도** | 1~2% 밀도, 본문 길이에 따라 5~6회 등장 | 기본 |
| **헤딩 구조** | 1,000자당 H2 3~5개, H1→H2→H3 계층 준수 | 기본 |
| **이미지** | 최소 3장(최적 6+), ALT 텍스트 20~50자에 키워드 포함 | 기본 |
| **해시태그** | 5~10개, 키워드 포함 필수 | 기본 |
| **가독성** | 평균 문장/문단 길이, 문장 길이 분산도, 구어체 비율 | 신규 |
| **경험 정보** | 경험 마커, 1인칭 대명사, 감정/감각 표현 (22개 패턴) | 신규 |
| **정보 충실성** | 리스트, 수치/통계, 비교/대조 표현, 고유명사 포함도 | 신규 |
| **AI 안전** | 반복 시작 패턴, 템플릿 문구, 문단 균일성, 구어체 검출 | 신규 |

등급 체계: A (90+), B (70~89), C (50~69), D (50 미만)

#### 3가지 SEO 프로파일

사용 목적에 따라 검증 가중치를 다르게 적용한다:

| 프로파일 | 용도 | 핵심 가중치 |
|---------|------|-----------|
| **balanced** (기본) | 범용 | 키워드 18%, 가독성 12%, AI안전 10% — 균형 배분 |
| **keyword_focused** | 상위 노출 목표 | 키워드 밀도 30%, 해시태그 15% — 키워드 집중 |
| **authenticity** | 저품질 판정 방지 | 경험정보 20%, AI안전 20% — 진정성 강조 |

같은 글이라도 프로파일에 따라 점수가 달라진다. 예를 들어 경험담이 풍부한 글은 `authenticity`에서 높은 점수를, 키워드가 촘촘한 글은 `keyword_focused`에서 높은 점수를 받는다.

#### 4가지 자동 최적화 전략

SEO 검증 결과에서 약점이 감지되면, 전략별 LLM 프롬프트로 글을 자동 개선한다:

| 전략 | 프롬프트 | 개선 초점 |
|------|---------|---------|
| **균형 최적화** | `seo_optimization.yaml` | 낮은 항목 전반 개선 |
| **정보 충실도** | `seo_strategy_depth.yaml` | 수치, 리스트, 비교/대조, 통계 데이터 추가 |
| **경험 주입** | `seo_strategy_experience.yaml` | 1인칭 서술, 감정 표현, 체험담, 시행착오 추가 |
| **AI 안전성** | `seo_strategy_human.yaml` | 반복 패턴 제거, 구어체 강화, 비대칭 구조 |

**최적화 플로우:**

```
초안 작성 완료
    │
    ▼
SEO 10항목 검증 (프로파일 선택)
    │
    ├── 약점 항목 자동 감지
    │
    ▼
최적화 전략 선택
    │
    ▼
LLM 스트리밍 재작성 (문체 유지)
    │
    ▼
전후 점수 비교 (delta 표시)
```

### 문제 3: 기존 블로그의 문체를 어떻게 유지할 것인가

LLM이 생성한 글은 기존 블로그 글과 톤이 다르면 독자에게 이질감을 준다. 하지만 매번 "이런 문체로 써줘"라고 지시하는 것은 일관성이 없다.

**해결: 레퍼런스 크롤링 + 프롬프트 주입**

```
기존 블로그 URL 3개 입력
    │
    ▼
BeautifulSoup 크롤링
    ├── 제목 + 본문 텍스트 (최대 1,500자)
    ├── 이미지 위치 배열 [paragraph_indices]
    └── JSON으로 저장 (data/references/)
    │
    ▼
초안 생성 시 system prompt에 주입
    ├── LLM이 문체 패턴 학습 (구어체, 문장 길이, 이모지 사용 등)
    └── 이미지 배치 패턴도 참조 → [IMAGE_N] 플레이스홀더 위치 결정
```

크롤러는 네이버 블로그의 DOM 구조 변경에 대비하여 **3단계 CSS 셀렉터 폴백**을 구현했다 (`.se-title-text` → `.pcol1` → `.se_title`). 크롤링이 실패하면 사용자가 직접 복사-붙여넣기할 수 있는 수동 입력 폴백도 지원한다.

SEO 최적화 단계에서도 레퍼런스가 다시 주입되어, 점수를 올리면서도 문체가 변하지 않도록 보장한다.

### 문제 4: 이미지를 자동으로 분석하고 최적 배치할 수 있는가

블로그 글에 이미지는 필수지만, 각 이미지에 대해 SEO에 적합한 설명과 ALT 텍스트를 수동으로 작성하는 것은 번거롭다.

**해결: Vision LLM 자동 분석 + 이미지 최적화 파이프라인**

```
이미지 업로드
    │
    ├── [자동 분석 모드] Vision LLM으로 이미지 설명 생성
    │   └── 타겟 키워드와 연관지어 30~80자 설명 자동 작성
    │
    └── [수동 입력 모드] 사용자가 직접 설명 작성
    │
    ▼
이미지 처리 파이프라인
    ├── 포맷 검증 (JPEG/PNG/WEBP/GIF)
    ├── 최대 너비 860px 리사이즈 (네이버 블로그 최적)
    ├── RGBA → RGB 변환, JPEG 85% 품질
    └── Data URI (Base64)로 HTML 내 인라인 삽입
    │
    ▼
ALT 텍스트 자동 생성
    └── 키워드 포함 20~50자, SEO 최적화
```

Vision 분석이 실패하면 UI에서 경고를 표시하고 수동 입력으로 폴백한다. 이미지 간격은 300~500자를 유지하여 가독성을 확보한다.

### 문제 5: 이미지를 포함한 글을 네이버에 자동 발행

네이버 블로그에 이미지가 포함된 HTML 글을 프로그래밍 방식으로 발행하려면 별도의 API 연동이 필요하다.

**해결: XML-RPC MetaWeblog API + Base64 이미지 임베딩**

```python
# MetaWeblog API 호출 구조
client.metaWeblog.newPost(
    blog_id, blog_id, api_secret,
    {
        "title": title,
        "description": html_with_images,
        "mt_keywords": "tag1,tag2,..."
    },
    True  # 즉시 발행
)
```

글의 `[IMAGE_N]` 플레이스홀더를 `<img src="data:image/jpeg;base64,..." />`로 치환하여 이미지를 인라인 삽입한다.

발행 외에 **3가지 내보내기 형식**도 지원한다:

| 형식 | 용도 |
|------|------|
| HTML | 네이버 블로그 에디터에 직접 붙여넣기 |
| 평문 | 다른 플랫폼 복사 |
| Markdown | Notion, GitHub, Jekyll 용도 |

Streamlit 6-페이지 구성:

```
[1. 설정] → [2. 키워드] → [3. 글쓰기] → [4. 미리보기] → [5. 이력] → [6. SEO 실험실]
    │           │              │              │             │            │
    │         시드 입력      이미지 업로드    SEO 검증      분석          프로파일 비교

    │         상위 포스트    초안 생성        전략 최적화    월별 추이     콘텐츠 테스트
    │         타겟 선택      템플릿 적용      내보내기/발행  CSV 다운로드  일괄 분석

  Ollama 모델 선택
  네이버 API 인증
  레퍼런스 URL 크롤링
```

## SEO 실험실

고급 사용자를 위한 전용 분석 도구로, 4개 탭으로 구성된다:

| 탭 | 기능 | 설명 |
|----|------|------|
| **프로파일 비교** | 3개 프로파일 동시 분석 | 항목별 점수 비교 테이블 + 바 차트 |
| **가중치 튜닝** | 10개 항목별 슬라이더 | 커스텀 가중치로 실시간 점수 재계산 |
| **콘텐츠 테스트** | HTML/평문 직접 입력 | 즉시 SEO 분석, 경쟁 블로그 벤치마킹 |
| **일괄 분석** | 이력 폴더 전체 재분석 | 월별 작성 수, 평균 점수, 약점 항목 감지, CSV 다운로드 |

## LLM 응답 안정성 처리

LLM이 JSON 형식의 응답을 생성할 때 마크다운 펜스(`` ```json ``)를 포함하거나 불완전한 JSON을 반환하는 경우가 빈번하다. 이를 위해 **3단계 파싱 전략**을 구현했다:
1. 마크다운 펜스 제거
2. JSON 추출 (중괄호 범위 탐색)
3. 부분 JSON 복구 (trailing comma 제거 등)
